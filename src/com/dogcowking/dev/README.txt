* 

JodaTime 이나 LocalDate 등 사용하지 않음.

JodaTime 이 기능들을 잘 구현하고 있음.
DateTime 객체에서 직접 월, 일 등을 추출해놓는다든지, Immutable 로 구현되었다든지 하는.

이 유틸은 그런 목적이 아님.
더 많은, 더 정확한 기능 구현이 아니라 그냥 구현시 간단하게 날짜 관련 코드를 다루는 것임.


*
Date 클래스를 그대로 사용해야 함.
기본기능 구현하되, 코드양을 줄일것. (이는 null 에 대한 처리도 포함됨)
다른 라이브러리에 의존성 갖지 않을것.
코드 일부를 복사해다가 써도 문제가 생기지 않을것. 
JAR 나 의존성관리툴로 가져다 쓸 수 있지만, 코드 자체를 가져다 쓸 수 있을것.
코드를 가져다 쓰기 위해 최대한 하나의 파일로 통일할것. 그렇지 못한 경우에 대한 
코드를 직접 보고 학습, 수정 가능할것. 이를 위해 동작방식은 주석처리 될것.
의존성 관리툴을 사용하지 않을것.
혹시나 별도의 의존성을 가진다면 관련 주석을 달고, 해당 기능은 다른 클래스로 분리될것.
구현과 주석은 초심자를 고려할 것. 더 Dck에 줄임말 선언한것 외에 더이상 여기저기서 선언, 상속하지 않을것.
비슷한 기능, 오버로딩된 메서드는 함께 둘것.



* '코드양을 줄일것' 에는 클래스 이름을 줄이는 것도 포함되는데,
안타깝게도 다른 여러 유틸도 추가하려하다보니 완벽하게 이뤄내지 못했음.

종전에는 Dck 클래스 하나로 표현되었지만, 명령의 종류별로 그룹을 두어야 할 필요를 느껴 결국
Dck
Dck.d (날짜 관련)
Dck.s (문자열 관련)
Dck.p (콘솔 프린트 관련)
